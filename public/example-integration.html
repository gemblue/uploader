⬆ Contoh Integrasi — Uploader.js

Dua pendekatan integrasi: App Key di client (dilindungi Origin Check) atau menggunakan Proxy Server (keamanan maksimal).

Dua pendekatan integrasi

Pilih sesuai kebutuhan keamanan aplikasi Anda:

Pendekatan A — App Key di client + Origin Check (Contoh 1–3) Browser (https://aplikasi-saya.com) │ POST sign-url.php │ app_key: APP_KEY_... # terlihat di source, tapi... │ Origin: https://aplikasi-saya.com # dikirim otomatis browser ▼ sign-url.php ├─ Origin cocok dengan allowed_origins? → ✔ buat token ├─ Rate limit belum habis? → ✔ lanjut └─ Domain lain / limit habis → 403 / 429 ✖ Pendekatan B — Proxy Server (Contoh 4) Browser → POST /api/minta-token (server Anda) # tanpa app key ↓ App Key di .env, tidak pernah ke browser POST sign-url.php → token → Browser
✅ Pendekatan A aman untuk kebanyakan kasus (SaaS, web app publik) — browser memblokir request dari domain yang tidak terdaftar.
✅ Pendekatan B untuk keamanan penuh — App Key tidak bisa disalahgunakan meski seseorang meng-copy source code halaman.

Contoh 1 — Form HTML (auto-bind) Origin Check

App Key boleh ada di HTML. Server membatasi penggunaan hanya dari domain yang terdaftar di allowed_origins — domain lain akan mendapat 403. Tambahkan data-app-key dan arahkan data-sign-url langsung ke sign-url.php. 

<!-- Di config.php, daftarkan domain Anda di allowed_origins -->

<form
  data-uploader
  data-sign-url="https://uploader.example.com/public/sign-url.php"
  data-app-key="APP_KEY_CONTOH_1"  <!-- terlindungi origin check -->
  data-folder="files/invoice"
  data-filename-prefix="inv"
>
  <input type="file" name="file" />
  <button type="submit">Upload</button>
</form>

<script src="uploader.js"></script>
<script>
  Uploader.bindForms();
  document.querySelector('form[data-uploader]')
    .addEventListener('uploader:success', (e) => {
      console.log('Berhasil:', e.detail.file);
    });
</script>

Contoh 2 — JavaScript API Origin Check

Kontrol penuh via JS. App Key disertakan langsung — server memvalidasi bahwa request berasal dari domain yang terdaftar. 

const uploader = new Uploader({
  signUrl: 'https://uploader.example.com/public/sign-url.php',
  appKey:  'APP_KEY_CONTOH_1',  // dilindungi origin check
  filenamePrefix: 'laporan',
});

await uploader.upload(file, {
  onSign:     (data) => console.log('Token:', data.token),
  onProgress: (pct)  => progressBar.value = pct,
  onSuccess:  (data) => alert('Berhasil: ' + data.file),
  onError:    (msg)  => alert('Error: ' + msg),
});

Contoh 3 — Hanya Minta Token Origin Check

Ambil token lebih awal (mis. saat user memilih file), lalu upload di langkah terpisah. Berguna untuk alur drag-and-drop, preview sebelum upload, atau multi-step form. 

const uploader = new Uploader({
  signUrl: 'https://uploader.example.com/public/sign-url.php',
  appKey:  'APP_KEY_CONTOH_1',
});

const signData = await uploader.requestToken();
// signData.token      — kirim ini saat upload
// signData.upload_url — URL upload.php
// signData.config     — konfigurasi aktif (folder, max_size, dll.)

Contoh 4 — Via Proxy Server Keamanan Maksimal

App Key tidak ada di client sama sekali. Proxy di server aplikasi Anda yang memegang App Key dan meneruskan permintaan token ke uploader server. Gunakan ini jika Anda membutuhkan lapisan otorisasi tambahan (cek login, CSRF, dll.) atau ingin App Key benar-benar tersembunyi. SDK PHP tersedia di sdk/UploaderSDK.php. 

// Di server aplikasi Anda (PHP, Laravel, CI, dst.)
// require 'UploaderSDK.php';
// $sdk = new UploaderSDK(getenv('UPLOADER_SIGN_URL'), getenv('UPLOADER_APP_KEY'));
// echo json_encode(UploaderSDK::publicPayload($sdk->requestToken()));

// Di client — tanpa app key
const uploader = new Uploader({
  signUrl: '/api/minta-token',  // proxy endpoint di server Anda
});
await uploader.upload(file, { onSuccess: (d) => console.log(d.file) });
